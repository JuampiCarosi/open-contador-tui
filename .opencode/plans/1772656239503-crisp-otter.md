# Plan: helper USD->ARS con Dólar MEP (por ítem)

## Objetivo
Agregar una ayuda en la creación de factura para cargar precio unitario en USD y convertirlo automáticamente a ARS usando Dólar MEP (DolarApi), asegurando que la factura emitida quede 100% pesificada (sin campos USD en payload final).

## Decisiones cerradas
- API a usar: `https://dolarapi.com/docs/`
- Endpoint MEP: `GET https://dolarapi.com/v1/dolares/bolsa`
- Campo de conversión USD->ARS: `compra`
- Alcance funcional: conversión **por ítem** (no total global en USD)

## Archivos críticos a modificar
- `src/services/dolar-mep-client.ts` (nuevo)
- `src/ui/state/invoice-draft.ts`
- `src/ui/screens/app-screen.ts`
- `src/services/sos-contador-client.ts`
- `.env.example`
- `README.md`

## Implementación recomendada

### 1) Cliente de cotización MEP (nuevo)
Crear `src/services/dolar-mep-client.ts` con estilo similar a `SosContadorClient`:
- Clase `DolarMepClient` + error tipado `DolarMepClientError`
- `fetch` con timeout (`AbortController`) y retries con backoff
- Cache en memoria con TTL para no pegar a la API en cada interacción
- Método principal `obtenerMep({ forceRefresh?: boolean })`
- Parse y validación defensiva de `compra`, `venta`, `fechaActualizacion`
- Respuesta normalizada para UI:
  - `compra`, `venta`, `fechaActualizacion`, `fetchedAt`, `fromCache`, `isStale`

Variables nuevas (con defaults seguros):
- `DOLAR_MEP_BASE_URL` (default `https://dolarapi.com/v1`)
- `DOLAR_MEP_TIMEOUT_MS` (default `4000`)
- `DOLAR_MEP_RETRIES` (default `1`)
- `DOLAR_MEP_CACHE_TTL_MS` (default `300000`)
- `DOLAR_MEP_MAX_AGE_MINUTES` (default `180`)

### 2) Estado del draft: helper USD transitorio
Extender `FacturaDraft` en `src/ui/state/invoice-draft.ts` con campos auxiliares (solo para la UI de carga):
- `modoCargaMoneda: "ARS" | "USD"`
- `precioUnitarioUsd: number`
- `cotizacionMep?: { compra: number; venta: number; fechaActualizacion: string; fetchedAt: number }`

Agregar helpers puros:
- `roundMoney(n)`
- `convertUsdToArs(usd, compra)`
- `isCotizacionStale(fechaActualizacion, maxAgeMinutes)`

Notas de diseño:
- `items` y `itemPendiente.precioUnitario` siguen en ARS (contable)
- Los campos USD no deben participar en `calcularTotales`

### 3) UI del wizard (step items)
En `src/ui/screens/app-screen.ts`, dentro del step `items`:
- Agregar campo/acción para toggle de moneda de carga (`ARS`/`USD`)
- Mostrar cotización MEP actual (`compra`) y timestamp
- Agregar acción `Refrescar cotización MEP`
- Si modo `USD`, editar `precioUnitarioUsd`; si modo `ARS`, mantener edición actual de `precioUnitario`

Al hacer `Agregar ítem`:
- Validar descripción/cantidad > 0
- Si modo ARS: flujo actual
- Si modo USD:
  - requiere cotización válida y no desactualizada
  - calcula `precioUnitarioArs = roundMoney(precioUnitarioUsd * compra)`
  - guarda el ítem en `draft.items` con `precioUnitario` en ARS
  - limpia `precioUnitarioUsd` y `itemPendiente`
- Mensajes claros de estado para éxito/error/falta de cotización

No cambiar navegación base (`↑/↓`, `Enter`, `Esc`) para minimizar riesgo.

### 4) Garantía de payload final pesificado
En `src/services/sos-contador-client.ts` (`crearFacturaDesdeDraft`):
- Mantener armado de `imputaciones`/`productos` exclusivamente desde `draft.items`
- Agregar saneo defensivo previo (normalizar numéricos) antes del `JSON.stringify`
- No incluir campos auxiliares USD en body

Resultado esperado: el backend recibe únicamente ARS como hoy.

### 5) Config y documentación
- Actualizar `.env.example` con `DOLAR_MEP_*`
- Actualizar `README.md`:
  - nuevas variables
  - explicar que USD es helper de carga y la factura se emite en ARS

## Edge cases cubiertos
- API MEP caída/timeout: bloquear solo carga USD; ARS sigue operativo
- Cache disponible pero stale: mostrar advertencia y forzar refresh para confirmar carga USD
- Sin cache + sin red: mensaje accionable para usar ARS o reintentar
- Redondeo monetario: 2 decimales en conversión por ítem

   - Verificar bloqueo de agregar ítem USD hasta refrescar
